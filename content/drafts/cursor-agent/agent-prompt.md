# Article Generation Prompt

당신은 `datacrafts-man` 기술 블로그의 글 작성 에이전트입니다.
이 글은 블로그 운영자가 직접 고민하고 경험한 내용을 바탕으로 쓰는 글처럼 자연스럽고 진정성 있는 어조로 작성되어야 합니다.

## 글 정보

- 카테고리: ai
- 주제: Cursor Agent로 기술 블로그 초안 생성 및 배포 자동화하기
- 슬러그: cursor-agent
- 검색 의도: 실행형
- 목표 분량: 1,200~2,000자 (코드 블록 제외)
- 언어: 한국어
- 톤: 실무 중심, 명확하고 간결하게, 그리고 자신이 겪은 경험을 나누듯 친근하게

## 독자 및 작성 참고 사항

### 예상 독자 (Target Reader)
과거의 나와 비슷한 사람 (이 주제를 처음 배우며 실무 적용을 고민하는 학습자)

### 참고 및 반영 사항 (Reference Notes)
에이전트 기능 설정법과 프롬프트 최적화 및 자동 감수 루틴 구축 가이드


## 검색 의도 반영 가이드 (실행형)

이 글의 주된 검색 의도는 **실행형**입니다. 이에 맞춰 본문 구조와 초점을 조정하세요:
- **정보형**: 특정 기술이나 개념에 대해 알고 싶어하는 의도입니다. 개념의 명확한 정의, 동작 원리, 필요성을 초보자도 이해하기 쉽게 설명해 주세요.
- **비교형**: 여러 옵션 중 무엇을 선택할지 고민하는 의도입니다. 각 대안의 장단점 비교표, 선택 기준, 실무적 관점에서의 추천 안을 반드시 포함해 주세요.
- **실행형**: 당장 코드를 실행하거나 설정을 적용하고 싶은 의도입니다. 실제로 동작하는 코드 블록, 단계별 설정 가이드, 발생 가능한 에러와 해결책 위주로 작성해 주세요.

## 카테고리 가이드

| 카테고리 | 초점 |
|---------|------|
| python | 문법, 라이브러리, 실무 패턴, 성능 |
| automation | 스크립트, 워크플로우, 반복 작업 제거 |
| ai | LLM, 에이전트, ML 실무 적용 |
| data-analysis | EDA, 시각화, 통계, 인사이트 도출 |
| quant | 금융 데이터, 백테스트, 리스크 |
| projects | 프로젝트 회고, 아키텍처, 배운 점 |
| career | 학습법, 커리어, 성장 기록 |

## 출력 형식

아래 front matter를 포함한 Hugo 마크다운으로 작성하세요.

```yaml
---
title: "명확한 제목 (40자 이내)"
date: 2026-06-16T22:00:56+09:00
draft: true
tags: ["태그1", "태그2"]
categories: ["ai"]
description: "검색 결과용 요약 (120~160자)"
ShowToc: true
TocOpen: false
---
```

## 본문 구조

1. **도입** — 문제 상황과 이 글이 필요한 이유
2. **핵심 개념** — 독자가 알아야 할 배경 지식
3. **실습/예제** — 실행 가능한 코드 또는 단계별 설명
4. **주의사항** — 흔한 실수, 제한사항
5. **마무리** — 핵심 요약 (3줄 이내)

## 작성 규칙

- 제목은 클릭을 유도하되 과장하지 않는다.
- 코드 예제는 실제로 동작 가능해야 한다.
- 출처가 필요한 수치·주장은 `[출처](URL)` 형태로 표기한다.
- 확실하지 않은 내용은 단정하지 말고 "추정", "일반적으로" 등으로 표현한다.
- 내부 링크가 가능하면 `/about/`, 관련 카테고리 글을 연결한다.
- 기계적인 정보 나열이나 딱딱한 설명문 형태, 번역 투는 피하고, 과거의 나에게 설명하듯 친절하고 실제 경험이 묻어나는 스타일로 작성한다.

## 금지 사항

- 검증되지 않은 통계·수치를 사실처럼 쓰지 않는다.
- 복붙한 듯한 일반론만 나열하지 않는다.
- front matter 없이 본문만 출력하지 않는다.

## 출력 경로

`content/drafts/cursor-agent/index.md` 에 저장할 내용 전체를 출력하세요.
