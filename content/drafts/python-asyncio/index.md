파이썬에서 비동기 프로그래밍을 처음 접할 때 가장 많이 하는 실수는 "비동기 코드를 작성하면 여러 개의 연산이 멀티스레드처럼 물리적으로 동시에 병렬 실행될 것"이라는 오해입니다. 결론부터 말씀드리면, **asyncio는 싱글 스레드에서 돌아가며, 대기 시간(I/O 바운드 작업)이 발생할 때 CPU가 놀지 않고 다른 작업을 하도록 스케줄링하는 동시성(Concurrency) 라이브러리**입니다.

과거의 저 역시 API 호출이 수백 개로 늘어나면서 프로그램이 멈춘 듯 느려지는 문제를 겪었고, 그때 처음으로 `asyncio`를 접했습니다. 처음 코드를 마주했을 땐 `async`, `await` 키워드와 이벤트 루프라는 개념이 마치 거대한 벽처럼 다가왔습니다. 이 글에서는 제가 비동기 프로그래밍을 배우며 겪었던 시행착오를 바탕으로, 핵심 개념인 코루틴과 이벤트 루프의 원리를 풀어서 설명하고, 동기 코드를 비동기로 전환하는 실전 예제를 나누고자 합니다.

---

## 1. 비동기 프로그래밍의 3대 핵심 개념 (코루틴, 태스크, 이벤트 루프)

asyncio를 이해하기 위해서는 아래 세 가지 주춧돌을 반드시 이해하고 넘어가야 합니다.

*   **코루틴 (Coroutine)**: 일반적인 함수(서브루틴)는 호출되면 처음부터 끝까지 한 번에 실행되지만, 코루틴은 중간에 실행을 일시 중단(suspend)하고 나중에 그 지점부터 다시 재개(resume)할 수 있는 특별한 함수입니다. 파이썬에서는 `async def`로 정의합니다.
*   **태스크 (Task)**: 코루틴을 이벤트 루프에 등록하여 예약하고 실행하는 단위입니다. 태스크를 통해 코루틴의 실행 상태를 추적하고 결과를 받아올 수 있습니다.
*   **이벤트 루프 (Event Loop)**: 비동기 애플리케이션의 심장입니다. 루프를 돌면서 대기 중인 코루틴들을 하나씩 실행하고, I/O 작업이 완료된 코루틴들을 재개하는 역할을 총괄합니다.

---

## 2. 동기와 비동기의 동작 방식 비교

동기식 코드와 비동기식 코드가 시간이 걸리는 작업을 처리할 때 어떻게 다른지 직관적으로 비교해 보겠습니다. 음식점에서 직원이 주문을 처리하는 과정에 비유하면 이해하기 쉽습니다.

| 비교 항목 | 동기 (Synchronous) | 비동기 (Asynchronous / asyncio) |
| :--- | :--- | :--- |
| **비유 (음식점)** | 손님이 주문한 음식이 나올 때까지 직원이 아무것도 안 하고 서서 기다림 | 음식을 주문하고 주방에 넘긴 뒤, 다음 손님의 주문을 받음 |
| **CPU 활용** | I/O 대기(네트워크, 파일 읽기 등) 동안 CPU가 대기 상태로 낭비됨 | I/O 대기 동안 CPU는 다른 예약된 코루틴(Task)을 처리함 |
| **코드 특징** | 순차적으로 실행되며 이전 라인이 끝나야 다음 라인 실행 | `await` 키워드를 만나면 실행권을 루프에 양도하고 대기함 |

---

## 3. 동기 코드를 비동기로 바꾸는 실전 예제

간단하게 웹 페이지 3개를 다운로드하는 작업을 가정해 보겠습니다. 실제 네트워크 요청 대신 `time.sleep`과 `asyncio.sleep`을 이용하여 시뮬레이션해 봅니다.

### 기존 동기식 코드
이 코드는 각 작업이 차례대로 실행되므로 총 3초가 걸립니다.

```python
import time

def download_page(page_id):
    print(f"페이지 {page_id} 다운로드 시작...")
    time.sleep(1)  # 1초 대기 (I/O 작업 시뮬레이션)
    print(f"페이지 {page_id} 다운로드 완료!")
    return f"콘텐츠 {page_id}"

def main():
    start = time.time()
    results = [download_page(i) for i in range(1, 4)]
    print(f"총 소요 시간: {time.time() - start:.2f}초")
    print(results)

if __name__ == "__main__":
    main()
```

### asyncio 비동기식 코드
`async`와 `await`을 적용하여 대기 시간 동안 다른 코루틴이 동작하게 만듭니다. 이 코드는 병렬적으로 대기가 이루어지므로 총 1초 내외가 소요됩니다.

```python
import asyncio
import time

async def download_page_async(page_id):
    print(f"페이지 {page_id} 다운로드 시작...")
    # 중요: time.sleep() 대신 asyncio.sleep()을 사용하여 루프에 제어권을 넘김
    await asyncio.sleep(1)
    print(f"페이지 {page_id} 다운로드 완료!")
    return f"콘텐츠 {page_id}"

async def main():
    start = time.time()
    # 3개의 코루틴을 Task로 묶어 동시에 실행 예약
    tasks = [download_page_async(i) for i in range(1, 4)]
    results = await asyncio.gather(*tasks)
    print(f"총 소요 시간: {time.time() - start:.2f}초")
    print(results)

if __name__ == "__main__":
    # 파이썬 3.7+ 에서 이벤트 루프를 생성하고 메인 코루틴을 실행하는 표준 방법
    asyncio.run(main())
```

## 4. 비동기 프로그래밍 시 자주 하는 실수와 주의사항

*   **블로킹 함수 사용**: 비동기 루프 안에서 `time.sleep()`이나 대용량 파일 동기 읽기, 동기식 `requests.get()`을 사용하면 이벤트 루프 자체가 통째로 블로킹되어 비동기의 이점이 사라집니다. 비동기 환경에서는 반드시 `asyncio.sleep()`이나 `aiohttp` 같은 비동기 라이브러리를 사용해야 합니다.
*   **await 키워드 누락**: `async def`로 만든 코루틴은 호출하는 것만으로는 실행되지 않고 코루틴 객체만 반환합니다. 반드시 앞에 `await`을 붙이거나 `asyncio.create_task()`로 루프에 스케줄링해야 합니다.
*   **CPU 바운드 작업에 사용**: `asyncio`는 파일 읽기/쓰기, 네트워크 요청 등 I/O 바운드 작업에 특화되어 있습니다. 소수 계산이나 대용량 이미지 처리 같은 CPU 바운드 작업은 `asyncio`를 써도 속도가 빨라지지 않으며, 오히려 이벤트 루프가 멈출 수 있습니다. 이 경우 `multiprocessing`이나 `concurrent.futures`를 사용해야 합니다.

## 5. 자주 묻는 질문 (FAQ)

### Q. asyncio를 쓰면 멀티코어 CPU를 활용하나요?
아닙니다. 파이썬의 `asyncio`는 GIL(Global Interpreter Lock) 하에서 단일 스레드로 동작합니다. 따라서 코어가 아무리 많아도 하나의 코어만 사용하며, 멀티코어를 활용해 병렬 연산을 수행하려면 `multiprocessing`을 함께 활용해야 합니다.

### Q. 일반 동기 함수 안에서 비동기 코루틴을 호출할 수 있나요?
일반 동기 함수 내에서는 직접 `await`을 사용할 수 없습니다. 동기 함수에서 비동기 코드를 실행하려면 `asyncio.run()`을 이용해 새로운 이벤트 루프를 실행하거나, 이미 실행 중인 이벤트 루프가 있다면 `asyncio.run_coroutine_threadsafe()` 등을 사용하여 코루틴을 전달해야 합니다.

### Q. 비동기 루프 안에서 동기식 라이브러리를 꼭 써야 할 때는 어떻게 하나요?
어쩔 수 없이 동기식 데이터베이스 드라이버나 블로킹 라이브러리를 사용해야 하는 경우, `asyncio.to_thread()` (파이썬 3.9+) 또는 `loop.run_in_executor()`를 사용하여 해당 동기 작업을 별도의 백그라운드 스레드풀에서 실행시켜 이벤트 루프가 멈추는 것을 방지할 수 있습니다.
