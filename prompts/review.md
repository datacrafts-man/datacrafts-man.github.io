# Review Checklist

감수자(본인)가 초안을 발행 전에 확인하는 체크리스트입니다.

## 구조

- [ ] 제목이 주제를 정확히 반영하는가?
- [ ] 도입에서 "왜 읽어야 하는지"가 명확한가?
- [ ] H2/H3 계층이 논리적인가?
- [ ] 마무리 요약이 핵심만 담고 있는가?

## 내용 품질

- [ ] 기술적 설명에 오류가 없는가?
- [ ] 코드 예제를 직접 실행해 봤는가?
- [ ] 과장·단정 표현이 없는가?
- [ ] 독자 수준(초급/중급)에 맞는가?

## front matter

- [ ] `title`, `date`, `description`이 채워져 있는가?
- [ ] `categories`가 올바른 섹션인가?
- [ ] `tags`가 2~5개로 적절한가?
- [ ] `draft: true` 상태인가? (발행 전)

## SEO / 가독성

- [ ] description이 120~160자인가?
- [ ] 첫 문단에 핵심 키워드가 자연스럽게 포함되는가?
- [ ] 코드 블록에 언어 태그가 있는가? (예: ```python)
- [ ] 이미지가 있다면 alt 텍스트가 있는가?

## 발행 전 최종

- [ ] factcheck.md 기준으로 사실 확인 완료
- [ ] seo_check.py 통과
- [ ] `draft: false`로 변경 후 publish
